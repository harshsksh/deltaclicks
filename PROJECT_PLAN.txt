1. Project Overview & Objectives
1.1  Client Summary
The client is a digital marketing agency requiring a professional, high-converting, fully static frontend website. This site is their primary digital storefront - it must communicate expertise, showcase services, build credibility, and generate leads through a contact form. There is no backend, no database, and no CMS. All content is managed through static data files, and the developer is the sole person who will make any future updates.

1.2  Business Objectives the Website Must Cover
• Digital marketing services: SEO, Social Media Marketing (SMM), Search Engine Marketing (SEM), Content Marketing, Email Marketing, Influencer Marketing, Online Advertising
• Digital branding and advertising across websites, social media channels, mobile apps, and all digital mediums
• Consultancy, strategy development, and campaign management for businesses, startups, and organizations
• Website and web application design, development, hosting, and maintenance services for clients
• Digital content creation including graphics, videos, blogs, articles, and multimedia for marketing purposes
• Brand identity development: logo design, graphic design, and corporate communication services
• Market research, data analysis, performance tracking, and reporting using analytics platforms
• Affiliate marketing, online lead generation, and performance-based marketing services
• Acting as intermediary, agent, and service provider for digital advertising platforms and media buying

3.2  Dynamic Routes
Two dynamic routes use Next.js slug-based routing. Their content is loaded from static data files at build time using generateStaticParams:
• /services/[slug] - reads from /data/services.js. The slug field in each service object maps to the URL.
• /portfolio/[slug] - reads from /data/portfolio.js. Each project has a unique slug.
• /blog/[slug] - reads from /data/blog.js. Each post has a unique slug.
• generateStaticParams() must be exported from these page files so Next.js can pre-render all paths at build time.

PHASE 1: ENVIRONMENT SETUP & PROJECT INITIALIZATION
Estimated Duration: 1 to 2 Days
Phase 1.1 - Prerequisites
Before creating the project, verify the following are installed on the development machine:
• Node.js 18.x or 20.x LTS - download from nodejs.org and verify: node -v
• npm - comes with Node.js - verify: npm -v
• Git - download from git-scm.com - configure: git config --global user.name and user.email
• VS Code - install extensions: Tailwind CSS IntelliSense, ESLint, Prettier
• A GitHub account - for version control and Vercel integration
• A Vercel account - sign up free at vercel.com using your GitHub account
• A Web3Forms account - sign up free at web3forms.com - get the Access Key for the client email

Phase 1.2 - Create the Next.js Project
Run this command in your terminal to scaffold the project:
npx create-next-app@latest agency-website --use-npm
Select these answers when prompted during setup:
• Would you like to use TypeScript? - No
• Would you like to use ESLint? - Yes
• Would you like to use Tailwind CSS? - Yes
• Would you like your code inside a src/ directory? - No
• Would you like to use App Router? - Yes
• Would you like to customize the import alias? - No

Phase 1.3 - Install All Required Packages
Navigate to the project folder and install all dependencies in one command:
cd agency-website
npm install framer-motion aos lucide-react @tailwindcss/typography
Then add the typography plugin to tailwind.config.js under the plugins array:
plugins: [require('@tailwindcss/typography')]
Phase 1.4 - Configure Tailwind CSS
• Open tailwind.config.js and add the brand color palette under theme.extend.colors
• Add fontFamily for the selected Google Font (e.g., Inter or Plus Jakarta Sans)
• Ensure the content array includes: './app/**/*.{js,jsx}' and './components/**/*.{js,jsx}'
• Add any custom animation keyframes if needed under theme.extend.animation

Phase 1.5 - Set Up Environment Variables
Create a .env.local file in the project root and add the Web3Forms key:
NEXT_PUBLIC_WEB3FORMS_KEY=your_web3forms_access_key_here
NOTE: .env.local must be listed in .gitignore - never commit it. When deploying to Vercel, add this same key manually in the project Settings > Environment Variables section of the Vercel dashboard.
Phase 1.6 - Initialize Git and Push to GitHub
• Run: git init inside the project folder
• Create a new repository on GitHub (private recommended)
• Add remote origin and push: git remote add origin <url> then git push -u origin main
• Go to vercel.com and import this GitHub repository - Vercel auto-detects Next.js and creates a deployment
• From this point, every git push to main will trigger an automatic Vercel deployment


PHASE 2: GLOBAL LAYOUT, THEME & BASE COMPONENTS
Estimated Duration: 2 to 3 Days
This phase builds everything that appears across multiple pages. No page-specific content is built here. Complete all components in this phase before moving to Phase 3.

Phase 2.1 - Root Layout (app/layout.js)
• Import the chosen Google Font using next/font/google and apply it to the body
• Set up default site metadata: title template, description, Open Graph image, Twitter card
• Import globals.css
• Create a Providers client component that initializes AOS inside useEffect - wrap children with it
• Structure: <html> > <body> > <Providers> > <Navbar> + {children} + <Footer>

Phase 2.2 - Global CSS (app/globals.css)
• Add @tailwind base, @tailwind components, @tailwind utilities
• Define CSS variables for brand colors under :root (primary, secondary, accent)
• Add: html { scroll-behavior: smooth }
• Style scrollbar using ::-webkit-scrollbar for a modern look
• Add global base styles for headings, links, paragraph line-height

Phase 2.3 - Navbar Component (components/layout/Navbar.jsx)
The Navbar is critical. Build it carefully with all required behaviors.
• Mark as use client to enable scroll detection and menu toggle state
• Logo: Next.js Image component - links to homepage - scale on hover
• Desktop links: Home, About, Services (dropdown), Portfolio, Blog, Contact
• Services mega dropdown: appears on hover, shows all 8 service pages with icons
• Sticky behavior: transparent on top, adds solid background + shadow on scroll using useEffect
• Active link: use usePathname() from next/navigation to bold or underline the current page link
• CTA button: Get a Quote - links to /contact - always visible, styled prominently
• Mobile: hamburger icon (Menu/X from Lucide React) toggles the mobile drawer menu
• MobileMenu component: full-width overlay with all links, accordion for services, and CTA button
• Wrap mobile menu with Framer Motion AnimatePresence for smooth open/close animation

Phase 2.4 - Footer Component (components/layout/Footer.jsx)
• 4-column grid: Column 1 = Logo + tagline + social icons, Column 2 = Quick Links, Column 3 = Services, Column 4 = Contact Info
• Quick links: Home, About, Portfolio, Blog, Contact, Privacy Policy, Terms
• Services links: all 8 individual service page URLs
• Contact info: email, phone, city/country, office hours
• Social icons: LinkedIn, Instagram, Facebook, Twitter/X, YouTube - use Lucide React icons
• Bottom bar: copyright text, Privacy Policy link, Terms link
• Responsive: stacks to single column on mobile

Phase 2.5 - Reusable UI Components (components/ui)
• Button.jsx - Props: variant (primary/secondary/outline/ghost), size (sm/md/lg), href, onClick, disabled. Include Framer Motion hover/tap animations.
• SectionHeading.jsx - Props: label (small tag above), title (main h2), subtitle (optional), align (center/left). Used on every page section.
• Card.jsx - Generic wrapper with rounded corners, shadow, hover lift (translateY -4px), and padding props.
• Badge.jsx - Small pill label for categories and service tags. Props: text, color variant.

Phase 2.6 - Animation Wrapper Components (components/animations)
• FadeIn.jsx - Framer Motion opacity 0 to 1 on viewport entry. Props: delay, duration.
• SlideUp.jsx - Framer Motion y: 40 to 0 + opacity on viewport entry. Props: delay.
• StaggerContainer.jsx - Wraps a list of children and staggers their animation with a 0.1s delay between each.
• CounterAnimation.jsx - Uses useInView to detect viewport entry, then animates a number from 0 to a target value over 2 seconds using useEffect and setInterval.

PHASE 5: WEB3FORMS CONTACT FORM INTEGRATION
Estimated Duration: 1 Day
Phase 5.1 - Web3Forms Setup
• Sign up at web3forms.com using the client's email address
• Copy the Access Key shown in the dashboard
• Add to .env.local: NEXT_PUBLIC_WEB3FORMS_KEY=your_key
• Add the same key in Vercel project Settings > Environment Variables (required for production)

Phase 5.2 - ContactForm Component Implementation
Build /components/forms/ContactForm.jsx with the following logic:
• Mark as use client - this component uses state and browser APIs
• Use useState to manage: formData object, isLoading boolean, isSuccess boolean, errorMessage string
• On form submit: preventDefault, set isLoading to true
• Send POST request to: https://api.web3forms.com/submit with Content-Type: application/json
• Request body must include: access_key (from env var), name, email, phone, service, message, subject (custom email subject line), botcheck (honeypot - always send as empty string or false)
• On success (response.data.success === true): set isSuccess to true, clear formData, hide form and show thank-you message
• On error: set errorMessage, show error below the submit button
• Always set isLoading to false in the finally block
• Disable the submit button and replace its text with a spinner icon while isLoading is true

Phase 5.3 - Form Validation
• Validate on blur (when each field loses focus) and on submit attempt
• Full Name: required, minimum 2 characters
• Email: required, validate with regex: /^[^\s@]+@[^\s@]+\.[^\s@]+$/
• Phone: optional, but if entered must have at least 10 digits
• Message: required, minimum 20 characters
• Service: required - must select an option other than the placeholder
• Show red inline error messages below each field that fails validation
• Submit button remains disabled until all required fields pass validation

Phase 6.2 - Image Optimization
• Always use Next.js <Image> component - never plain <img> tags
• Every image must have: width, height, and alt attributes
• Set priority on hero images (above the fold) to preload them
• Compress all images to under 200KB before adding to /public - use squoosh.app
• Use WebP format for all project and blog images
• Use the sizes prop for responsive images to serve the correct size per breakpoint

Phase 6.3 - Performance Optimization
• Default: all components are React Server Components - only add use client when needed for state or browser APIs
• Use next/dynamic with { ssr: false } for any heavy third-party client-side components
• Google Fonts are already optimized via next/font - no additional work needed
• Target Lighthouse scores: Performance 90+, Accessibility 95+, Best Practices 95+, SEO 100
• Avoid importing entire icon libraries - import individual icons: import { ArrowRight } from 'lucide-react'

Phase 6.4 - Accessibility (a11y)
• Use semantic HTML throughout: <header>, <nav>, <main>, <section>, <article>, <footer>, <h1> through <h6>
• All images must have meaningful, descriptive alt text
• All form inputs must have a visible <label> with htmlFor matching the input's id
• All interactive elements must be keyboard accessible - verify by tabbing through the entire site
• Color contrast: all body text must meet WCAG AA - minimum 4.5:1 ratio against its background
• Icon buttons with no visible text must have aria-label attributes (e.g., aria-label='Close menu')
• Focus styles must be visible on all interactive elements - do not remove them with outline: none without a replacement

14. Developer Notes, Best Practices & Timeline

14.1 - Coding Standards
• Component naming: PascalCase (e.g., HeroSection.jsx, ContactForm.jsx)
• Variable and function naming: camelCase (e.g., handleSubmit, isLoading, formData)
• Every component file has a single default export
• Keep components under 150 lines - split into sub-components if they grow larger
• Write comments above complex logic: API calls, animation triggers, dynamic data patterns

14.2 - Tailwind Best Practices
• Design mobile-first: write base classes for mobile, add sm:, md:, lg: prefixes to scale up
• Avoid arbitrary values like w-[347px] - use the design system's spacing scale
• Extract repeated long class strings into components or @apply rules in globals.css
• Do not mix inline style and Tailwind for the same property on the same element

14.3 - Next.js App Router Important Rules
• Default: every component is a Server Component - only add 'use client' when you need useState, useEffect, event listeners, or browser APIs
• use client components cannot be imported directly into server components - wrap them in a parent client component if needed
• Always export generateStaticParams from dynamic route pages ([slug]) for static pre-rendering
• Use loading.js and error.js files in route folders to handle loading states and errors gracefully

14.4 - Git Workflow Recommendation
• Commit at the end of every sub-phase with a clear message (e.g., 'feat: build testimonials slider component')
• Use feature branches per phase: git checkout -b phase/3-pages
• Open a Pull Request per phase - Vercel auto-generates a preview URL per PR for client review
• Merge to main only after client approves the preview deployment
